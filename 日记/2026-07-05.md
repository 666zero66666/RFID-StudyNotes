# Obsidian 插件使用指南与配置建议

> 适用于当前 RFID vault 已安装 / 已启用的插件。插件调整后建议重启 Obsidian 或执行 `Reload app without saving`。

## 1. Claudian

用途：在 vault 内直接让 AI 帮你整理笔记、生成模板、改写内容、检查链接和管理 Markdown 文件。

常用方式：

- 选中一段文字后让 Claudian 总结、改写或提取问题。
- 让 Claudian 新建结构化笔记，例如论文笔记、读书笔记、导览页。
- 让 Claudian 检查某个文件夹内的链接、标题和结构。

配置建议：

- 保持启用即可。
- 对重要笔记操作前，可以要求“先备份再修改”。

## 2. Calendar

用途：提供日历视图，快速打开每日笔记。

当前配置配合：

- 每日笔记文件夹：`日记/`
- 每日笔记模板：[[模板/每日笔记.md]]

使用方式：

- 点击右侧日历中的日期，打开或创建当天日记。
- 适合记录每日学习进度、论文阅读进度、实验记录入口。

配置建议：

- Settings → Daily notes：确认 New file location 是 `日记`。
- Date format 推荐：`YYYY-MM-DD`。

## 3. Dataview

用途：把笔记当作数据库查询，自动生成列表和表格。

典型用途：

- 自动列出所有论文笔记。
- 自动列出最近修改的笔记。
- 自动汇总某类实验记录或问题追踪。

示例：

```dataview
TABLE file.mtime AS 最近修改
FROM ""
WHERE type = "paper-note" OR contains(file.tags, "paper")
SORT file.mtime DESC
```

配置建议：

- 保持默认即可。
- 后续模板中的 `type` 字段要尽量保持一致，例如 `paper-note`、`experiment-log`、`issue-tracker`。

## 4. PDF++

用途：增强 PDF 阅读、标注和与 Markdown 笔记的联动。

适合场景：

- 阅读 RFID 标准 PDF。
- 阅读论文 PDF。
- 从 PDF 标注跳转到对应笔记。

配置建议：

- Settings → PDF++ 中先保持默认。
- 建议把 PDF 和对应笔记放在同一主题文件夹中，例如 `入门资料2/翻译/`。
- 对重要 PDF 建议单独建一篇读书/论文笔记链接它。

## 5. Templater

用途：比 Obsidian 内置 Templates 更强的模板插件，可以自动插入日期、标题等动态内容。

当前模板文件夹：[[模板/模板说明.md]]

常用命令：

- `Templater: Open Insert Template modal`
- `Templater: Replace templates in the active file`

使用建议：

- 新建笔记后，用 Templater 插入：[[模板/读书笔记.md]]、[[模板/论文阅读笔记.md]]、[[模板/RFID协议概念卡.md]]。
- 模板里的 `<% tp.date.now(...) %>` 需要通过 Templater 插入才会自动展开。

配置建议：

- Settings → Templater → Template folder location：`模板`
- 安全起见，System commands 保持关闭。

## 6. Tasks

用途：增强待办事项管理，支持截止日期、优先级、循环任务和查询。

基本写法：

```markdown
- [ ] 阅读 Select 命令 #task 📅 2026-07-06
- [ ] 整理 Session/S2 概念卡 #task 🔼
```

查询示例：

```tasks
not done
sort by due
sort by priority
```

配置建议：

- Settings → Tasks 中可以保持默认。
- 建议所有正式待办都加 `#task`，方便过滤。
- 截止日期用 Tasks 插件命令插入，减少格式错误。

## 7. Style Settings

用途：给主题和部分插件提供图形化样式设置。

使用方式：

- Settings → Style Settings。
- 如果当前主题或插件支持 Style Settings，就可以在里面调字体、颜色、间距等。

配置建议：

- 初期不需要大改。
- 如果以后安装 Minimal、AnuPpuccin 等主题，可以用它统一微调样式。

## 8. Iconize

用途：给文件夹和笔记添加图标，让导航更直观。

推荐图标规划：

- `日记`：日历图标
- `模板`：工具 / 模板图标
- `入门资料2`：书本图标
- `任务导览.md`：地图 / 指南图标

配置建议：

- 在文件或文件夹上右键，找 Iconize / Change icon 相关操作。
- 图标只用于视觉导航，不影响笔记内容。

## 9. Hider

用途：隐藏 Obsidian 界面里暂时不需要的元素，让界面更清爽。

适合隐藏：

- 不常用的 ribbon 图标。
- 状态栏中的冗余信息。
- 标题栏或部分 UI 元素。

配置建议：

- 不建议一次隐藏太多。
- 如果找不到某个按钮，先检查 Hider 是否隐藏了它。

## 10. QuickAdd

用途：快速创建指定类型笔记，适合做“新建论文笔记”“新建实验记录”等自动化入口。

推荐以后配置的 QuickAdd 动作：

- 新建论文阅读笔记 → 使用 [[模板/论文阅读笔记.md]]
- 新建 RFID 概念卡 → 使用 [[模板/RFID协议概念卡.md]]
- 新建实验记录 → 使用 [[模板/实验记录.md]]
- 新建问题追踪 → 使用 [[模板/问题追踪.md]]

配置路径：

- Settings → QuickAdd → Add Choice。
- Choice 类型可先用 Template。
- Template path 选择对应模板。

配置建议：

- 等模板使用稳定后，再配置 QuickAdd 自动化。
- 先不要做太复杂的脚本，避免维护成本过高。

## 11. Advanced Tables

用途：增强 Markdown 表格编辑。

常用场景：

- 整理协议字段表。
- 整理论文对比表。
- 整理实验参数表。

使用建议：

- 在表格中按 Tab / Enter 辅助跳格和新建行。
- 用命令面板搜索 `Advanced Tables` 查看可用命令。

配置建议：

- 默认即可。
- 如果表格格式混乱，可以用它的格式化表格命令。

## 12. Omnisearch

用途：比 Obsidian 默认搜索更强的全文搜索。

使用方式：

- 命令面板搜索 `Omnisearch`。
- 搜索 C1G2、Select、Query、Session 等关键词。

配置建议：

- 首次使用可能需要建立索引。
- 如果搜索不到新内容，等几秒或重启 Obsidian。

## 13. Recent Files

用途：显示最近打开的文件，便于快速回到刚刚读过的笔记。

使用建议：

- 适合配合论文阅读和翻译阅读。
- 如果侧边栏出现 Recent Files 面板，可以固定在右侧。

配置建议：

- 默认即可。

## 14. Obsidian 内置 Templates

用途：插入静态模板。

当前配置：

- 模板目录：[[模板/模板说明.md]]
- 日期格式：`YYYY-MM-DD`
- 时间格式：`HH:mm`

使用方式：

- 命令面板：`Templates: Insert template`

注意：

- 内置 Templates 不会执行 Templater 的 `<% ... %>` 动态语法。
- 如果要自动生成日期，优先用 Templater 插入模板。

## 15. Multi-Column Markdown

用途：分栏阅读 / 多栏排版。适合阅读长文、标准文档、翻译稿，也可以把某篇笔记在阅读模式下显示成 2 栏或 3 栏。

当前安装版本：`0.9.1`

### 方式 A：整篇笔记分栏阅读

在笔记最上方 frontmatter 中加入：

```yaml
---
Multi-Column Markdown:
  - Number of columns: 2
  - Border: off
---
```

如果想三栏阅读，可以改成：

```yaml
---
Multi-Column Markdown:
  - Number of columns: 3
  - Border: off
---
```

注意：

- 整篇文档分栏主要在 Reading mode / 阅读模式中生效。
- 修改分栏设置后，如果没有立刻变化，可以关闭再打开该笔记，或重载 Obsidian。
- 很长的段落可能不会被拆开到不同栏，这是插件和 Obsidian 渲染机制的限制。
- PDF 导出不一定支持分栏效果。

### 方式 B：用命令自动插入分栏配置

打开命令面板，搜索：

```text
Setup Multi-Column Reflow
```

这个命令可以帮助生成正确的 YAML/frontmatter 分栏配置。

### 推荐用法

- 阅读 [[入门资料2/翻译/C1G2_2015_全文翻译.md]] 这类长文时，可以先复制一份或在读书笔记中使用 2 栏。
- 对正文翻译原文件，建议谨慎添加分栏 frontmatter；如果只想改善阅读体验，可以优先在读书笔记里分栏。
- 2 栏适合 MacBook / 普通屏幕；3 栏更适合外接大屏。


## 16. Inline Annotations：给 Markdown 某一部分做批注

用途：给文章中的某个词、某句话或某一小段直接添加“行内批注”。阅读时点击被批注的文字，就会弹出批注内容；也可以右键编辑或删除。

当前安装版本：`1.0.0`

### 最推荐的使用流程

1. 在编辑模式中选中要批注的文字。
2. 按快捷键：`Cmd + Shift + A`。
3. 输入批注内容，按 `Ctrl + Enter` 或点击 Save。
4. 切换到阅读模式或继续在编辑模式中，点击被标记的文字查看批注。
5. 需要修改/删除时，右键被批注文字，选择 `Edit annotation` 或 `Remove annotation`。

### 它在 Markdown 里实际长什么样

插件会把：

```markdown
Select 命令
```

变成：

```markdown
{Select 命令::这里写我的批注，例如：这是 C1G2 中最重要的集合筛选工具。}
```

阅读时只显示 `Select 命令`，点击后弹出后面的批注内容。

### 增删改查怎么实现

| 操作 | 方法 |
|---|---|
| 增加批注 | 选中文字 → `Cmd + Shift + A` → 输入批注 |
| 查看批注 | 点击被批注文字；也可以在插件设置里改为 hover 悬停显示 |
| 修改批注 | 右键被批注文字 → `Edit annotation` |
| 删除批注 | 右键被批注文字 → `Remove annotation` |
| 搜索批注 | Obsidian 搜索 `::关键词` 或直接搜索批注文字 |

### 与普通链接的区别

- 普通双链 `[[某篇笔记]]` 更适合“文章与文章之间”的连接。
- Inline Annotations 更适合“某句话 / 某个术语 / 某个段落旁边的批注”。
- 它不会额外创建一篇批注笔记，而是把批注直接保存在当前 Markdown 文件中。

### 适合用在哪里

- 阅读 [[入门资料2/翻译/C1G2_2015_全文翻译.md]] 时，对某个术语直接写理解。
- 阅读论文翻译时，对算法步骤、公式、实验结果做即时评论。
- 写作时标记“这里需要补证据”“这里以后要重写”。



### Inline Annotations 不显示 / 快捷键没反应的排查

如果插件装了但在 Obsidian 插件列表中看不到，或 `Cmd + Shift + A` 没反应，优先检查这几点：

1. 插件文件是否存在：`.obsidian/plugins/inline-annotations/manifest.json`。
2. 启用列表 `.obsidian/community-plugins.json` 中是否包含：`inline-annotations`。
3. 修改配置文件后，需要重启 Obsidian，或执行命令：`Reload app without saving`。
4. 只有插件真正加载后，快捷键 `Cmd + Shift + A` 才会出现在 Hotkeys 中并生效。
5. 如果 Obsidian 正在运行，有时它会用当前内存中的插件列表覆盖手动写入的配置；这种情况下重新写入后一定要立刻重载 Obsidian。

本次问题原因：插件文件已安装，快捷键也已写入，但 `inline-annotations` 没在 `community-plugins.json` 启用列表里，所以 Obsidian 没加载它。已重新加入启用列表。

### 注意事项

- 批注语法是 `{原文::批注}`，所以如果原文或批注中有很多 `{}`、`::`，可能会影响解析。
- 很长的大段批注建议单独建笔记，再用普通链接连接；Inline Annotations 适合短批注。
- 如果想要像 Word 那样集中管理所有批注，后续可以再安装更重的侧边栏批注类插件。

# 如何链接到 RFID vault 外的文件夹？

结论：可以，但要区分两种链接。

## 1. Wiki 链接不能直接链接 vault 外部

Obsidian 的 Wiki 链接主要用于当前 vault 内部：

```markdown
[[入门资料2/翻译/C1G2_2015_全文翻译.md]]
```

这种写法不能可靠地指向 RFID 文件夹外面的文件或文件夹。

## 2. 链接外部文件 / 文件夹，用 Markdown 的 file 链接

如果要打开 RFID vault 外部的文件夹，可以写成：

```markdown
[打开桌面](file:///Users/thcao/Desktop/)
```

如果要打开某个外部文件，可以写成：

```markdown
[打开外部 PDF](file:///Users/thcao/Desktop/example.pdf)
```

如果路径里有空格，建议把空格写成 `%20`，例如：

```markdown
[打开外部文件夹](file:///Users/thcao/Desktop/My%20Folder/)
```

## 3. 外部链接的限制

- 外部文件不会进入 Obsidian 的反向链接、图谱和 Dataview 管理。
- 换电脑后，绝对路径可能失效。
- macOS / Obsidian 可能会弹出安全确认。
- 不能像 vault 内部笔记一样被 Wiki 双链自动维护。

## 4. 推荐做法

- 如果只是偶尔打开外部资料：用 `file:///...` 链接即可。
- 如果资料需要长期阅读、搜索、反链和整理：建议复制或移动到 RFID vault 内。
- 可以单独建一篇“外部资料索引”笔记，集中放外部文件夹链接。
