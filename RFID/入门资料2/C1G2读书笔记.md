盘点标志（Inventoried flag）

指示标签是否可以响应读写器的标志。标签在四个 session 中各⾃维护⼀个独⽴的 inventoried flag；每个标志都

具有对称的 A 和 B 两个取值。在任⼀给定 session 内，读写器通常先将标签从 A 盘点到 B，再将标签从 B 重新盘

点回 A（反之亦可）。

---

Q

读写器⽤于调节标签响应概率的参数。读写器在⼀个盘点轮次中指示标签将⼀个 Q 位随机数或伪随机数装⼊其时隙

计数器；读写器也可以命令标签递减其时隙计数器。当时隙计数器的值即 slot 为 0 时，标签回复。Q 是⼀个位于 0 

到 15 范围内的整数，对应的标签响应概率范围是从 2^0 = 1 到 2^-15 = 0.000031 。

----

读写器通过三类基本操作管理标签群体：

a) Select。选择标签群体。读写器可以利⽤ Select 命令基于标签存储器中的⼀个或多个值选择⼀个或多个标签，也

可以利⽤ Challenge 命令基于标签是否⽀持所需密码套件与认证类型来对⼀个或多个标签发起挑战。随后读写器可

以盘点并访问被选中的标签。

b) Inventory。识别单个标签。读写器通过在四个 session 中之⼀发送 Query 命令来开始⼀个盘点轮次。⼀个或多

个标签可能会回复。读写器检测单标签回复，并请求该标签的 EPC。Inventory 由多个命令组成。⼀个盘点轮次在

任意时刻只能⼯作在⼀个 session 中。

c) Access。与已识别标签通信。读写器可以执⾏核⼼操作，例如读取、写⼊、锁定或 kill 标签；也可以执⾏与安全

相关的操作，例如认证标签；还可以执⾏与⽂件相关的操作，例如打开标签⽤户存储区中的某个特定⽂件。Access 

由多个命令组成。读写器只能访问已经被唯⼀识别出的标签。

表 6.16：链路定时参数

T1

最⼩值： MAX(RTcal,10Tpri) × (1 - |FrT|) - 2µs

标称值： MAX(RTcal,10Tpri)

最⼤值： MAX(RTcal,10Tpri) × (1 + |FrT|) + 2µs

含义：从读写器发射到标签⽴即回复之间的时间。更具体地说，是从读写器传输最后⼀位最后⼀个上升沿，到

⽴即标签回复第⼀个上升沿之间的时间，在标签天线端测量。

T2

最⼩值： 3.0Tpri

最⼤值： 20.0Tpri

含义：如果标签要解调读写器信号，则这是读写器回复时间，测量点为标签回复最后⼀个（虚拟）⽐特结束到

读写器传输第⼀个下降沿之间的时间。

T3

最⼩值： 0.0Tpri

含义：在 T1 之后，读写器等待再发出下⼀条命令的时间。

T4

最⼩值： 2.0 RTcal

含义：读写器命令之间的最⼩时间。

T5

最⼩值： MAX(RTcal,10Tpri) × (1 - |FrT|) - 2µs

最⼤值： 20ms

含义：从读写器发射到标签延迟回复之间的时间。更具体地说，是从读写器传输最后⼀位最后⼀个上升沿，到

延迟标签回复第⼀个上升沿之间的时间，在标签天线端测量。

T6

最⼩值： MAX(RTcal,10Tpri) × (1 - |FrT|) - 2µs

最⼤值： 20ms

含义：从读写器发射到第⼀次处理中标签回复之间的时间。更具体地说，是从读写器传输最后⼀位最后⼀个上

升沿，到第⼀条标签回复第⼀个上升沿之间的时间；该回复表明标签要么仍在处理，要么已经处理完成。

T7

最⼩值： MAX(250µs, T2(max))

最⼤值： 20ms

含义：处理中标签各次回复之间的时间。更具体地说，是从前⼀次“仍在处理”的标签传输最后⼀个虚拟⽐特结

束，到当前标签回复第⼀个上升沿之间的时间；该回复表明标签要么仍在处理，要么已经完成。

标签应以下列两种⽅式之⼀计算并存储 StoredCRC，具体由标签制造商决定：

1. 当读写器写⼊或覆盖 EPC 中的⽐特时计算并更新；

2. 每次标签上电时计算并更新。

Select 过程由两条命令组成，即 Select 和 Challenge。Select 允许读写器为后续盘点选择⼀个标签群体；

Challenge 允许读写器为后续认证对⼀个标签群体发起挑战。Select 与 Challenge 是读写器在开始盘点前可以发出

的仅有两种命令，⽽且这两种命令并不互斥，也就是读写器可以在开始⼀轮盘点前同时发出 Select 和 Challenge。

Select 是强制命令；Challenge 是可选命令。

Select 包含以下参数： Target 、 Action 、 MemBank 、 Pointer 、 Length 、 Mask 和 Truncate
