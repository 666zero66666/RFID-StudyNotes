## 0. 本章总览

信源编码解决的是：**如何把信源产生的信息转换成适合数字通信系统传输的比特流，并尽量降低码率。**

对于模拟信源，信源编码通常包括两个层次：

1. **模 / 数转换**：把模拟信号变成数字信号；
2. **压缩编码**：去除冗余，降低传输码率。

模 / 数转换的基本流程是：

```text
模拟信号 → 采样 → 量化 → 编码 → 数字信号
```

其中：

| 步骤 | 解决的问题 | 结果 |
|---|---|---|
| 采样 | 时间连续怎么办？ | 时间离散化 |
| 量化 | 幅度连续怎么办？ | 幅度离散化 |
| 编码 | 离散电平如何传输？ | 二进制比特流 |

# 4.1 模 / 数转换

## 一、采样

### 1. 采样的作用

采样是在发送端以固定时间间隔对模拟信号取值，将连续时间信号变成离散时间样值序列。

也就是说，采样完成的是：

> **时间轴上的离散化。**

采样后，信号不再是每一时刻都有值，而是只在一系列采样时刻有值。

### 2. 从时域理解采样

从时域看，采样可以理解为：

- 用一系列等间隔的采样时刻去“截取”原模拟信号；
- 得到一串样值；
- 如果采样频率足够高，这些样值可以代表原始信号的变化趋势。

理想情况下，采样信号可以看成模拟信号与冲激序列相乘得到的一系列冲激样值。

### 3. 从频域理解采样

从频域看，采样会使原始信号频谱按照采样频率 $f_s$ 为间隔进行周期性拓展。

这点很重要，因为能否恢复原始信号，本质上取决于：

> 周期拓展后的频谱之间是否发生重叠。

如果不重叠，可以用低通滤波器恢复原信号；如果重叠，就会发生频率混叠。

### 4. 信号重建原理

接收端可以用理想低通滤波器从采样信号中恢复模拟信号。

有两种理解方式：

| 角度 | 理解 |
|---|---|
| 时域 | 每个采样冲激通过理想低通滤波器后形成 sinc 脉冲，所有 sinc 脉冲叠加恢复原波形 |
| 频域 | 采样信号频谱与理想低通滤波器频率响应相乘，滤出原始频谱 |

所以，采样不是随便取几个点，而是有严格条件的。

### 5. 奈奎斯特采样定理

为了保证能从采样信号中恢复出原始模拟信号，采样频率必须大于模拟信号最高频率的 2 倍：

$$
f_s > 2f_{max}
$$

其中：

| 符号 | 含义 |
|---|---|
| $f_s$ | 采样频率 |
| $f_{max}$ | 模拟信号最高频率 |

这就是奈奎斯特采样定理。

例如电话语音信号最高频率约为 $3400\text{Hz}$，则采样频率应大于：

$$
2\times3400=6800\text{Hz}
$$

实际 PCM 语音通常采用 $8\text{kHz}$ 采样频率，因此满足采样定理。

### 6. 为什么要求大于 2 倍

如果采样频率足够高，频域中周期拓展后的频谱之间会留有间隔，接收端可以用低通滤波器取出原始频谱。

如果采样频率逐渐降低，拓展频谱之间的间隔会变小。

当采样频率低于信号最高频率的 2 倍时，不同周期的频谱会发生重叠，接收端就无法判断某个频率成分到底来自原信号的哪个部分。

### 7. 频率混叠

当采样频率不满足采样定理时，会发生**频率混叠**。

频率混叠是指：

> 采样后，高频信号可能被错误地表现为低频信号。

典型例子是电影中的车轮反转现象：

- 电影帧率相当于采样频率；
- 车轮旋转频率越来越高；
- 当旋转频率超过帧率能正确表示的范围时，人眼看到的车轮可能变慢甚至反转。

这不是车轮真的反转，而是采样导致的混叠假象。

### 8. 理想采样与实际采样

| 类型 | 特点 |
|---|---|
| 理想采样 | 用冲激信号与模拟信号相乘，得到一系列冲激样值 |
| 实际采样 | 不必真的生成冲激信号，只需要取得采样时刻的电平值 |

实际系统中，采样通常由 ADC 的采样保持电路完成。

## 二、量化

### 1. 量化的作用

采样后，信号已经在时间上离散，但每个采样值的幅度仍然可以连续变化。

量化的作用是：

> 把连续幅度的采样值映射到有限个量化电平上，实现幅度离散化。

因此：

- 采样解决“时间连续”的问题；
- 量化解决“幅度连续”的问题。

### 2. 量化基本概念

| 概念 | 含义 |
|---|---|
| 量化级数 | 量化电平的个数 |
| 量化间隔 | 相邻量化电平之间的距离 |
| 量化误差 | 实际信号电平与量化后电平之间的差值 |
| 量化噪声 | 量化误差也称为量化噪声 |
| 量化信噪比 | 信号功率与量化噪声功率之比 |

量化误差最大通常为量化间隔的一半。

量化级数越多，量化间隔越小，量化误差越小，但需要的编码比特数也越多。

### 3. 均匀量化

均匀量化是指：

> 量化电平等间隔分布。

优点：

- 原理简单；
- 实现方便；
- 编码关系清晰。

缺点：

- 对小信号不友好；
- 小信号幅度本来就小，量化噪声相对占比大，量化信噪比较低。

### 4. 非均匀量化

非均匀量化是指：

> 量化电平不等间隔，小信号细量化，大信号粗量化。

具体表现为：

| 信号幅度 | 量化间隔 | 目的 |
|---|---|---|
| 小信号 | 小 | 减小相对量化误差，提高小信号信噪比 |
| 大信号 | 大 | 允许较大误差，节省量化级数 |

电话语音中，小信号出现概率较高，而且电话通信要求一定的信噪比，因此非均匀量化很有意义。

### 5. 压缩与扩张

实际系统中，非均匀量化常通过“压缩 + 均匀量化”实现。

发送端：

```text
模拟信号 → 压缩器 → 均匀量化器
```

接收端：

```text
解码信号 → 扩张器 → 近似恢复原信号
```

压缩器的作用是：

- 小信号相对放大；
- 大信号相对压缩。

这样再进行均匀量化时，小信号可以获得更好的量化精度。

## 三、编码

### 1. 编码的作用

编码是将量化后的电平值用二进制数字表示。

例如，如果量化后只有有限个电平，那么每个电平都可以分配一个二进制码字。

### 2. 编码位数

如果量化电平数为 $N$，则每个量化值需要：

$$
\log_2N
$$

位二进制数字表示。

举例：

| 量化级数 | 编码位数 |
|---|---|
| 16 | 4 bit |
| 256 | 8 bit |
| 8192 | 13 bit |
| 65536 | 16 bit |

### 3. 编码后的结果

经过采样、量化、编码后，模拟信号就变成了数字比特流，可以进入后续的信道编码、调制和传输环节。

---

## 四、实现

### 1. ADC

ADC 是 Analog-to-Digital Converter，即模 / 数转换器。

它完成：

```text
模拟电压 → 数字码字
```

并行比较型 ADC 的典型组成包括：

- 电阻分压器；
- 电压比较器；
- 寄存器；
- 编码器。

工作过程：

1. 电阻分压器产生多个参考电压；
2. 输入电压与参考电压比较；
3. 比较结果被寄存器保存；
4. 编码器把比较结果转换成二进制输出。

并行比较型 ADC 速度快，但比较器数量较多。

### 2. DAC

DAC 是 Digital-to-Analog Converter，即数 / 模转换器。

它完成：

```text
数字码字 → 模拟电压
```

典型的 $R-2R$ 网络 DAC 使用不同二进制位控制电子开关，使不同权重的电流参与求和，再由运算放大器输出模拟电压。

DAC 的核心思想是：

> 不同二进制位代表不同权重，最终输出电压与输入数字量成比例。

# 4.2 音频编码

## 一、G.711 PCM

G.711 PCM 是传统电话语音中典型的 PCM 编码方式。

| 项目 | 内容 |
|---|---|
| 音频范围 | $300\sim3400\text{Hz}$ |
| 采样频率 | $8\text{kHz}$ |
| 量化方式 | 非均匀量化 |
| 量化级数 | 256 |
| 编码字长 | 8 bit |
| 压缩 | 无 |
| 码率 | $8\text{kHz}\times8\text{bit}=64\text{kbit/s}$ |

关键点：

- 电话语音最高频率约 $3400\text{Hz}$；
- $8\text{kHz}$ 采样满足奈奎斯特采样定理；
- 8 bit 编码对应 256 个量化电平；
- 不进行压缩，所以码率就是 64 kbit/s。

---

## 二、RPE-LPT

RPE-LPT 是 GSM 早期采用的语音编码算法，全称为规则脉冲激励线性预测编码。

| 项目 | 内容 |
|---|---|
| 音频范围 | $300\sim3400\text{Hz}$ |
| 采样频率 | $8\text{kHz}$ |
| 量化方式 | 均匀量化 |
| 量化级数 | 8192 |
| 编码字长 | 13 bit |
| 原始码率 | $8\text{kHz}\times13\text{bit}=104\text{kbit/s}$ |
| 压缩率 | 8:1 |
| 压缩后码率 | $104/8=13\text{kbit/s}$ |

关键点：

- RPE-LPT 相比 G.711 增加了压缩；
- 原始码率为 104 kbit/s；
- 压缩后降到 13 kbit/s，更适合无线移动通信。

---

## 三、AMR-NB

AMR-NB 是 Narrow Band AMR，即窄带自适应多速率语音编码，常用于 3G WCDMA。

| 项目 | 内容 |
|---|---|
| 语音范围 | $300\sim3400\text{Hz}$ |
| 采样频率 | $8\text{kHz}$ |
| 量化方式 | 均匀量化 |
| 量化级数 | 65536 |
| 编码字长 | 16 bit |
| 原始码率 | $8\text{kHz}\times16\text{bit}=128\text{kbit/s}$ |
| 压缩方式 | 根据信道条件自适应调整压缩比 |
| 压缩后码率 | 4.75 / 5.15 / 5.90 / 6.70 / 7.40 / 7.95 / 10.2 / 12.2 kbit/s |

关键点：

- NB 表示窄带，语音频率范围仍是 $300\sim3400\text{Hz}$；
- AMR 的重点是自适应：信道好时提高码率，信道差时降低码率；
- 最高码率约 12.2 kbit/s。

---

## 四、AMR-WB

AMR-WB 是 Wide Band AMR，即宽带自适应多速率语音编码，常用于 4G VoLTE。

| 项目 | 内容 |
|---|---|
| 音频范围 | $50\sim7000\text{Hz}$ |
| 采样频率 | $16\text{kHz}$ |
| 量化方式 | 均匀量化 |
| 量化级数 | 65536 |
| 编码字长 | 16 bit |
| 原始码率 | $16\text{kHz}\times16\text{bit}=256\text{kbit/s}$ |
| 压缩方式 | 根据信道条件自适应调整压缩比 |
| 压缩后码率 | 6.60 / 8.85 / 12.65 / 14.25 / 15.85 / 18.25 / 19.85 / 23.05 / 23.85 kbit/s |

关键点：

- WB 表示宽带，频率范围扩展到 $50\sim7000\text{Hz}$；
- 采样频率提高到 $16\text{kHz}$；
- 声音听感比窄带语音更自然；
- 最高码率约 23.85 kbit/s。

---

## 音频编码对比

| 编码方式 | 采样率 | 编码字长 | 原始码率 | 压缩后码率 | 典型应用 |
|---|---:|---:|---:|---:|---|
| G.711 PCM | 8 kHz | 8 bit | 64 kbit/s | 64 kbit/s | 传统电话 |
| RPE-LPT | 8 kHz | 13 bit | 104 kbit/s | 13 kbit/s | GSM |
| AMR-NB | 8 kHz | 16 bit | 128 kbit/s | 最高 12.2 kbit/s | 3G 语音 |
| AMR-WB | 16 kHz | 16 bit | 256 kbit/s | 最高 23.85 kbit/s | VoLTE 高清语音 |

# 4.3 视频编码

## 一、概述

视频编码与音频编码类似，也包括：

```text
视频信号 → 模 / 数转换 → 压缩编码 → 数字视频码流
```

但视频信号比音频信号复杂得多，因为视频同时包含：

- 时间维度：一帧一帧变化；
- 空间维度：每一帧由大量像素组成；
- 色彩维度：每个像素还可能包含多种颜色分量。

因此，未经压缩的视频码率非常高，必须进行压缩。

### 1. 黑白、灰度和彩色视频

| 类型 | 每个像素表示方式 |
|---|---|
| 黑白图像 | 1 bit，表示黑或白 |
| 灰度图像 | 多级灰度，例如 8 bit 表示 256 级灰度 |
| 彩色图像 | 通常用 RGB 三个颜色分量表示 |

### 2. 彩色成像基本思想

彩色视频可以根据三基色原理，用红、绿、蓝三个分量表示。

需要注意：

> 光传感器本身不能直接感知“颜色”，只能感知光强。彩色图像通常需要把光分解成不同颜色分量，再分别采样。

---

## 二、模 / 数转换

视频的模 / 数转换包括采样、量化和编码。

### 1. 视频采样

视频采样有两个维度：

| 采样类型 | 含义 |
|---|---|
| 时间采样 | 每隔一定时间采一幅图像，形成一帧一帧的视频 |
| 空间采样 | 在水平方向和垂直方向采样，把图像离散成像素点 |

### 2. 空间采样与分辨率

图像在水平和垂直方向采样后形成像素矩阵。

视频分辨率通常表示为：

$$
\text{水平像素数}\times\text{垂直像素数}
$$

例如：

| 类型 | 分辨率示例 |
|---|---|
| PAL 制式 VCD | $352\times288$ |
| PAL 制式 DVD | $720\times576$ |
| 720P | $1280\times720$ |
| 1080P | $1920\times1080$ |
| 4K | $3840\times2160$ |
| 8K | $7680\times4320$ |

分辨率越高，空间采样点越多，画面越清晰，但数据量也越大。

### 3. 时间采样与帧率

视频是连续图像序列。

每秒显示的帧数称为帧率，单位为 FPS。

常见帧率包括：

- 24 FPS；
- 25 FPS；
- 30 FPS；
- 60 FPS。

人眼有视觉暂留现象，所以当帧率足够高时，一张张静态图像会被感知为连续运动。

### 4. 黑白图像码率

黑白图像每个像素只需要 1 bit。

以 $720\times480$、30 FPS 为例：

一帧数据量：

$$
720\times480\times1=345.6\text{kbit}
$$

视频码率：

$$
345.6\times30=10368\text{kbit/s}=10.4\text{Mbit/s}
$$

### 5. 灰度图像码率

如果采用 256 级灰度，每个像素需要 8 bit。

以 $720\times480$、30 FPS 为例：

一帧数据量：

$$
720\times480\times8=2764.8\text{kbit}
$$

视频码率：

$$
2764.8\times30=82944\text{kbit/s}=82.9\text{Mbit/s}
$$

### 6. 彩色图像码率

如果 RGB 三个分量各用 8 bit，则每个像素需要：

$$
8+8+8=24\text{bit}
$$

以 $720\times480$、30 FPS 为例：

一帧数据量：

$$
720\times480\times24=8.29\text{Mbit}
$$

视频码率：

$$
8.29\times30=248.7\text{Mbit/s}
$$

这个码率非常高，直接传输不现实，所以视频必须压缩。

### 7. 视频码率公式

未压缩视频码率可以概括为：

$$
\text{视频码率}=\text{分辨率}\times\text{帧率}\times\text{每像素编码比特数}
$$

也就是说，视频码率与以下因素成正比：

- 分辨率；
- 帧率；
- 每像素比特数。

---

## 三、视频压缩

### 1. 为什么要压缩

视频未经压缩时码率极高。

例如 $720\times480$、30 FPS、RGB 24 bit 彩色视频，码率约为：

$$
248.7\text{Mbit/s}
$$

这对普通通信系统来说开销过大，因此必须压缩。

视频压缩的核心思想是：

> 利用人眼感知特点和视频内容冗余，用更少的比特表示尽可能相同的视觉信息。

### 2. 色彩空间压缩

RGB 每个像素要保存红、绿、蓝三个分量。

但人眼对亮度更敏感，对色度不太敏感。因此可以把 RGB 转换成 YCbCr：

| 分量 | 含义 |
|---|---|
| Y | 亮度信号 |
| Cb | 蓝色色差信号 |
| Cr | 红色色差信号 |

压缩方式：

- 亮度 Y：保持较高采样率；
- 色度 Cb、Cr：降低采样率。

例如：

- RGB：每个像素 24 bit；
- YCbCr 降低色度采样后：平均每个像素约 16 bit；
- 相当于压缩约 33%。

### 3. 可变帧率压缩

VFR 是 Variable Frame Rate，即可变帧率。

基本思想：

| 场景 | 帧率策略 |
|---|---|
| 动态画面 | 使用较高帧率，保证流畅 |
| 静态画面 | 使用较低帧率，减少冗余 |

这样可以在主观视觉质量变化不大的情况下减小数据量。

### 4. 视频图像压缩

视频图像压缩算法可以分为两类：

| 类型 | 思想 | 代表 |
|---|---|---|
| 帧内压缩 | 每一帧单独压缩，不考虑帧间关系 | M-JPEG |
| 帧间压缩 | 利用相邻帧相似性，只编码变化部分 | MPEG |

### 5. M-JPEG 压缩

M-JPEG 是 Motion JPEG。

它的特点是：

- 每一帧都像 JPEG 图片一样单独压缩；
- 不考虑前后帧之间的相似性；
- 实现简单；
- 压缩效率不如 MPEG。

### 6. MPEG 压缩

MPEG 会利用相邻帧之间的相似性。

例如视频中背景不变，只有人物在移动，那么不必每一帧都完整编码背景，只需要编码变化部分。

这就是帧间压缩的核心思想。

### 7. I 帧、P 帧和 B 帧

MPEG 中常见三种帧：

| 帧类型 | 名称 | 特点 |
|---|---|---|
| I 帧 | 帧内编码帧 / 关键帧 | 完整编码图像，可独立解码 |
| P 帧 | 前向预测编码帧 | 参考前面的 I 帧或 P 帧，只编码差异 |
| B 帧 | 双向预测编码帧 | 同时参考前后帧，只编码差异 |

对比：

- I 帧数据量大，但最独立；
- P 帧数据量较小，但依赖前面的参考帧；
- B 帧压缩效率更高，但依赖前后帧，解码时延更大。

### 8. 运动补偿

如果物体在画面中移动，仅仅做帧间差分仍可能产生较多数据。

运动补偿的基本思想是：

1. 把图像分成多个小块；
2. 在参考帧中寻找相似块；
3. 如果找到了相似块，就只记录该块的运动信息，而不是重新编码整个块。

这样可以大幅降低运动视频的数据量。

### 9. 视频压缩标准

常见 MPEG 标准包括：

- MPEG-1；
- MPEG-2；
- MPEG-4。

H.264 又称 MPEG-4 Part 10 或 AVC，是常见的高级视频编码标准。

H.264 的压缩率很高，例如压缩后的 480P 视频码率可以远低于原始未压缩码率。

# 易混点

| 易混点 | 正确理解 |
|---|---|
| 采样和量化 | 采样是时间离散化，量化是幅度离散化 |
| 量化噪声和信道噪声 | 量化噪声来自数字化近似，信道噪声来自传输过程 |
| 均匀量化和非均匀量化 | 均匀量化电平等间隔，非均匀量化小信号细、大信号粗 |
| PCM 和压缩编码 | PCM 是基础数字化方式，不一定压缩；AMR、RPE-LPT 等会压缩 |
| M-JPEG 和 MPEG | M-JPEG 每帧独立压缩，MPEG 利用帧间相关性压缩 |
| I 帧、P 帧、B 帧 | I 帧独立，P 帧参考前帧，B 帧参考前后帧 |

# 与通信系统的关系

信源编码主要解决“**如何更高效地表示信息**”。

它与下一章[[05 第5章 信道编码与交织]]的区别是：

| 编码类型 | 目标 |
|---|---|
| 信源编码 | 减少冗余，降低码率，提高信息表示效率 |
| 信道编码 | 增加冗余，提高抗误码能力，保证传输可靠性 |

所以：

- 信源编码通常是在“压缩”；
- 信道编码通常是在“加冗余”；
- 二者目的不同，但都是数字通信系统中的关键环节。
